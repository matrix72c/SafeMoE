## Integration Check

Integration was checked directly against the current codebase because the dedicated `gsd-integration-checker` agent was unavailable in this session.

### Findings

1. **Data -> Training is wired**
   - `safemoe/pretrain.py` obtains `val_loaders_for_eval = data.val_dataloaders()` and creates split iterators from `data.get_loader(label)`.
   - This matches the Phase 1 export contract and the Phase 3 training loop expectations.

2. **Model/Masking -> Training is wired**
   - `HarmfulParamRegistry` is constructed in `safemoe/pretrain.py` before model wrapping.
   - The loop uses `random.choices(...)` for split selection, `activation_masker.enable()/disable()` around `D_std`, and `gradient_masker.mask(split_label)` for non-`D_unlabeled` steps.

3. **Training -> Evaluation is wired**
   - `evaluate_with_ablation()` is called from the save-interval path in `fit()` when `registry` and `val_loaders` are present.
   - `safemoe/evaluate.py` imports `validate()` from `safemoe.pretrain` and uses the same validation semantics as training.

4. **Checkpoint -> Ablation -> Evaluation is wired**
   - `safemoe/ablate.py` uses `HarmfulParamRegistry` to zero `theta_harmful`.
   - `safemoe/evaluate.py` writes `results.json` and `routing_attribution.json`.
   - `safemoe/__main__.py` exposes `pretrain`, `ablate`, and `evaluate` in `PARSER_DATA`.
